SemGaze Reason-Alignment (BGE) Handoff Notes
Date: 2026-02-15

Goal
- Use gaze-follow reason txt files as supervision.
- Convert each txt to a 768-dim feature using HuggingFace BGE (default: BAAI/bge-base-en-v1.5).
- Add an alignment loss between BGE feature and the gaze decoder's 1st-layer output (configurable).
- Enable alignment path ONLY for Train split (Val/Test unaffected).
- If feature missing or load fails: use 0 vector and reason_valid=0, plus print warnings (rate-limited).

New Script: BGE Feature Extraction (Precompute)
- File:
  semgaze/tools/extract_reason_features.py

- What it does:
  - Reads txt files from <input_root>/<split>/**/*.txt
  - Parses "Object:" and "Reasoning:" lines
  - text_mode:
    - object_reasoning: embed "Object: ...\nReasoning: ..."
    - object_only: embed only Object
    - reasoning_only: embed only Reasoning
  - Encodes using HF AutoModel + mean pooling + L2 normalize
  - Saves float32 tensors to <output_root>/<split>/**/<same_name>.pt

- Command example (train):
  python /home/elicer/semgaze/tools/extract_reason_features.py \
    --split train \
    --input_root /home/elicer/semgaze/data/bucket_data/data/gazefollow_reason/output \
    --output_root /home/elicer/semgaze/data/bucket_data/data/gazefollow_reason_bge/features \
    --model_name BAAI/bge-base-en-v1.5 \
    --text_mode object_reasoning

- Notes:
  - Use --device cpu if running on cheap CPU-only box (slow).
  - HF model download requires internet or pre-baked cache.

How Alignment Works (Implementation Details)
- Predicted alignment feature:
  - Take decoder intermediate "queries" at layer_index (1-based) from TwoWayTransformer
  - Remove not-a-person token
  - Select last person token (annotated person): align_feat_pred[:, -1, :]
  - Project to 768 with alignment_head and normalize

- Ground truth feature:
  - batch["reason_emb"] loaded from .pt and normalized in dataset
  - batch["reason_valid"] gates alignment loss (0 => contributes nothing)

- Alignment loss types:
  - cosine: mean(1 - cos(pred, gt))
  - mse: mean squared error
  - infonce: batch-wise symmetric cross entropy over pred<->gt, with align_logit_scale

Important Caveats
- config dataclasses in semgaze/semgaze/config.py are not updated; training uses Hydra cfg but type hints may be stale.
- If you load an old checkpoint via cfg.model.weights, strict=False is used; new alignment head weights will be randomly initialized.
- Alignment path is train-only by design (dataset + forward logic), so val/test will ignore reason features.
